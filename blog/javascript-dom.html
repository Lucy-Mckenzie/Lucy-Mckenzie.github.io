An analogy to describe JavaScript and its relationship to HTML and CSS.

I imagine a spy or superhero movie when movies were first made using stop motion and eventually short videos of everyday life. One of the most renowned early examples is Georges Méliès’ A Trip to the Moon, released in 1902—the tale of a group of astronomers who travel to the moon, encountering adventures along the way. The film features groundbreaking special effects, including innovative use of miniatures, hand-painted sets, and visual trickery.

Nowadays, movies are on a whole other level, using visual and practical effects, complex simulations of physics, weather, computer-generated imagery, and events that would be impossible to film in real life. To name a few.

This is where JavaScript comes in for websites, just like CGI transformed movies, JavaScript took the web to a new level. Before JavaScript, websites were like posters, pretty but static. With JavaScript, they became like video games: interactive and dynamic, responding to what you do.

Explain control flow and loops using an example process from everyday life

Control flow works like our everyday life, going from start to finish, following a step to step process to get things done.

For example:

- Start: check if you have dirty clothes.

- If yes: load the washing machine.

- And turn on.

- Wait for clothes to wash.

- Hang up washing.

- If no.

- You don't need to do washing today.

- The end. 


      function hasDirtyClothes() {
        if (dirtyClothes.length > 0) {
        loadWashingMachine()
        waitForCycle()
        hangUpWashing()
        } else {
        console.log("You don’t need to do washing today")
        }
      }

As you go through your daily life everyday, you are essentially doing the same habits (loops) on repeat. In the same way control flow goes down the list executing the statements, and when it comes to a loop such as the one above that returns false, it simply does nothing and keeps making its way down to the bottom.

Describe what the DOM is and an example of how you might interact with it

The DOM (Document Object Model) is essentially a structured, hierarchical representation of your HTML page. It's like a tree where every HTML object becomes a node (imagine them as branches containing the elements in your HTML) that can be accessed and manipulated with javascript.

For example:

HTML

 

    <button id="button">This is a button</button>
    <p id="text">This is the original text</p>

Javascript

 

    const button = document.getElementById("button");
    const text = document.getElementById("text");
    
    button.addEventListener("click", () => {
    
    text.textContent = "This text has been changed!";
    });

Explain the difference between accessing data from arrays and objects

Both arrays and objects store data whether it be strings or numbers, this helps us to condense and organise our code.

To understand how we can access data from either arrays or objects, its important to know how the data is stored

Array

An array is like a shelf, Imagine a book shelf where the books are placed in specific order, every book has a fixed position or index starting from 0. To get a book you have the know the exact position. 


                           // 0 //            // 1 //             // 2 //
    const books = ["The diamond sniper", "The rose code", "The alice network"];
    let book = book[1];  // ”The rose code”

Say you want the second book on the shelf, you would ask for book 1 as arrays are 0 indexed.

Object

An object is like a filling cabinet where each book is labelled instead of numbered, to access something you dont need to know the order, you just need to know the key or label.


  let bookCabinet = {
    Russia: "The diamond sniper",
    England: "The rose code",
    France: "The alice network"
    }
    
    let book = bookCabinet["Russia"];

So using the key you can then get out the variable stored inside of it.
